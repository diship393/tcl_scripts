1. Introduction

This report presents a performance benchmark of the SelectDCD feature in NAMD/VMD, evaluated using the Minimal Cell Model 200nm testbed (Zenodo record 17524049).

The Minimal Cell Model 200nm is a large-scale biological system containing 6,733,164 atoms distributed across 98 distinct molecular segments (segnames), including 93 protein/complex segments and 5 lipid membrane components (CHOL, DOPG, POPC, SSM, TOCL). Segment sizes range from 294 atoms (235_mon) to 2,070,837 atoms (CHOL). The system's substantial size makes it an ideal candidate for evaluating the performance benefits of selective trajectory loading versus traditional full-system approaches.

The SelectDCD feature enables NAMD to write per-segment DCD trajectory files during simulation, allowing downstream VMD analysis to load only the subset of atoms relevant to each computation. This avoids the overhead of loading the entire multi-million atom system when only a fraction of the coordinates are needed.

2. Benchmark Description

Test Protocol

Two experimental conditions were evaluated, each performing identical RMSD analysis across all 98 segments:

Set 1 — Full System (benchmark.dat)
The entire system is loaded once from a single PSF/DCD pair. For each of the 98 segments, an atom selection is created by segname and RMSD is computed over all trajectory frames. The initial load time is incurred once (28.6 s), but every RMSD calculation must iterate over the full coordinate arrays.
Set 2 — SelectDCD / Per-Segment (benchmark_dcdselect.dat)
Each segment has its own dedicated PSF and DCD files (produced by SelectDCD during simulation). For each segment, the per-segment PSF/DCD is loaded independently and RMSD is computed. This incurs a small load cost per segment but operates on a much smaller coordinate set.

Metrics Collected

For each segment the following quantities were recorded:

  • Load time — time to load the system (single 28.6 s event for Set 1; per-segment for Set 2)
  • RMSD analysis time — wall-clock time to compute RMSD across all frames
  • Memory delta — incremental memory allocated for the RMSD computation
  • Total memory — cumulative process memory at the end of each segment’s analysis

3. Benchmark Scripts

Both benchmarks were executed in VMD using Tcl scripts (vmd -dispdev text -e <script>).

Set 1 — Full System (compute_rmsd.tcl)

compute_rmsd.tcl 
# Open output file for RMSD values
set outfile [open "benchmark.dat" w]

set t0 [clock milliseconds]
set mem0 [lindex [exec ps -o rss= -p [pid]] 0]
# Load PSF and DCD files
mol new envelope.tpr
mol addfile traj.dcd waitfor all

set t1 [clock milliseconds]
set elapsed [expr {($t1 - $t0) / 1000.0}]
puts "Load System: ${elapsed}s"
puts $outfile "Load System: ${elapsed}s"

# Split by segname and write separate PSF/DCD per segment
set sel [atomselect top all]
set segnameList [lsort -unique [$sel get segname]]
$sel delete

# Get number of frames
set nframes [molinfo top get numframes]

set all [atomselect top all]

foreach segname $segnameList {
    set t0 [clock milliseconds]

    # Define protein selection for RMSD calculation
    set ref [atomselect top "segname $segname" frame 0]
    set sel [atomselect top "segname $segname"]
    # Compute RMSD for each frame
    for {set i 0} {$i < $nframes} {incr i} {
        $sel frame $i

        # Align to reference frame
        $all move [measure fit $sel $ref]

        # Compute RMSD
        set rmsd [measure rmsd $sel $ref]
    }
    $sel delete

    set t1 [clock milliseconds]
    set elapsed [expr {($t1 - $t0) / 1000.0}]

    set mem1 [lindex [exec ps -o rss= -p [pid]] 0]
    set mem_used [expr {($mem1 - $mem0) / 1024.0}]
    set mem_total [expr {$mem1 / 1024.0}]
    puts "segname $segname: ${elapsed}s | delta ... MB | total ... MB"
    puts $outfile "segname $segname: ${elapsed}s | delta ... MB | total ... MB"
}
close $outfile

This script loads the entire system once, then iterates over all unique segnames, performing alignment and RMSD computation against frame 0. The full coordinate set ($all) is moved during alignment, which means every frame update touches all atoms in memory.

Set 2 — SelectDCD / Per-Segment (compute_rmsd_dcdselect.tcl)

compute_rmsd_dcdselect.tcl 
set outfile [open "benchmark_dcdselect.dat" w]

mol new envelope.tpr
# Split by segname and write separate PSF/DCD per segment
set sel [atomselect top all]
set segnameList [lsort -unique [$sel get segname]]
$sel delete
mol delete all

foreach segname $segnameList {
    set t0 [clock milliseconds]
    set mem0 [lindex [exec ps -o rss= -p [pid]] 0]

    mol new $segname.psf
    mol addfile $segname.dcd waitfor all

    # Get number of frames
    set nframes [molinfo top get numframes]

    set t1 [clock milliseconds]
    set elapsed [expr {($t1 - $t0) / 1000.0}]
    puts "Load System $segname: ${elapsed}s"
    puts $outfile "Load System $segname: ${elapsed}s"

    set t0 [clock milliseconds]

    # Define protein selection for RMSD calculation
    set ref [atomselect top "segname $segname" frame 0]
    set sel [atomselect top "segname $segname"]
    # Compute RMSD for each frame
    for {set i 0} {$i < $nframes} {incr i} {
        $sel frame $i

        # Align to reference frame
        $sel move [measure fit $sel $ref]

        # Compute RMSD
        set rmsd [measure rmsd $sel $ref]
    }
    $sel delete

    set t1 [clock milliseconds]
    set elapsed [expr {($t1 - $t0) / 1000.0}]

    set mem1 [lindex [exec ps -o rss= -p [pid]] 0]
    set mem_used [expr {($mem1 - $mem0) / 1024.0}]
    set mem_total [expr {$mem1 / 1024.0}]
    puts "segname $segname: ${elapsed}s | delta ... MB | total ... MB"
    puts $outfile "segname $segname: ${elapsed}s | delta ... MB | total ... MB"
}

close $outfile

This script first reads the full topology to extract the list of segnames, then deletes the molecule and iterates over each segment, loading its dedicated PSF/DCD pair produced by SelectDCD. A key difference: alignment moves only the segment atoms ($sel move) rather than the entire system ($all move), which is both correct for isolated segments and significantly faster.

Although SelectDCD requires loading 98 separate files (67.6 s total vs 28.6 s for the full system), the dramatic reduction in RMSD computation time (22.7 s vs 324.0 s) more than compensates, yielding a 3.9× overall speedup.

Per-Segment RMSD Analysis

In the full-system approach, RMSD analysis takes approximately 3.0–3.1 s per protein segment — a nearly constant overhead dominated by iterating over the full coordinate arrays to extract the selection. The five lipid segments (which contain more atoms) take 5–10 s each.

With SelectDCD, small monomeric segments complete in 6–45 ms, while the largest lipid segments (CHOL, TOCL) take 5–7 s. The RMSD speedup ranges from 1.4× (CHOL, the largest segment) to 510× (875_mon, a small monomer).

Memory Usage

The full-system approach allocates ~22 GB immediately upon loading the PSF/DCD, then grows incrementally (~25.7 MB per protein segment) as RMSD selections are created. The SelectDCD approach starts at ~1.4 GB and grows gradually, with large jumps only for the lipid segments (CHOL alone adds 6.5 GB). For workflows analyzing only a subset of segments, SelectDCD would use dramatically less memory.

5. Time to Solution vs Segment Size

This figure reveals the key insight: in the full-system approach, RMSD time is nearly independent of segment size — a 294-atom monomer takes the same ~3 s as a 57,000-atom complex, because VMD must iterate over all 6.7 million coordinates to evaluate the atom selection. With SelectDCD, time scales proportionally with segment size, from 6 ms (469 atoms) to 7 s (2.07 M atoms). The crossover point where SelectDCD becomes slower than the amortized full-system approach occurs around ~1.5 M atoms.

8. Conclusions

3.9× total speedup (90.3 s vs 352.6 s) for a complete per-segment RMSD analysis of the Minimal Cell Model 200nm, despite needing to load 98 separate files.
14.2× RMSD analysis speedup (22.7 s vs 324.0 s), because each computation operates on a minimal coordinate set rather than scanning the full ~24 GB system.
Small protein segments benefit the most, with RMSD speedups of 100–510×. These represent the majority of the system (93 of 98 segnames).
Large lipid segments show modest benefit (CHOL: 1.4×, TOCL: 1.7×). Their per-segment DCD files are already large, so the coordinate-filtering advantage is smaller.
Memory usage is substantially lower for partial analyses. If a user only needs to analyze a subset of segments, SelectDCD avoids loading the full 24 GB system. For the first protein segment, memory is only 1.4 GB vs 22 GB.
Even with amortized load costs, SelectDCD is faster for 96 of 98 segments, confirming that the benefit stems primarily from reduced RMSD computation time rather than avoiding repeated full-system loads.

SelectDCD is particularly well-suited for workflows involving large multi-component systems where analysis targets individual molecular species — precisely the scenario encountered in minimal cell models and other complex biological assemblies.
